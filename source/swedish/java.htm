ToonTalk(TM) och Java(TM)

ToonTalk kan göra en Java applet från ett ToonTalk program.

För att översätta något till en Java applet ska man plocka upp det, trycka på pause-tangenten och sedan klicka på "Komma tillbaka senare". Det du gjort då är, att du flyttat det från ToonTalk in i Windows. Om allt fungerar ska en webbläsare öppnas automatiskt och du kommer att få se ditt program bli uppspelat. För att du själv ska kunna prova detta kan du gå till anteckningsboken och slå upp sidan 6, anteckningsboken för exempel, och på sidan 2 plocka upp roboten som hela tiden lägger till 1. Tryck på pause-tangenten och klicka på "Komma tillbaka senare". När din applet dyker upp i din webbläsare, klicka på "Start" och du kan se hur siffrorna blir större. Det är bara robotar, lastbilar och bilder som kan skapa en Java applet om de plockas upp medan ToonTalk stannas. En robot kommer köras med en låda som är likadan som den han tränades med. När du laddar upp en lastbil behöver du kanske trycka på F8 för att hindra den från att köra bort. Du kan lägga ett textkort i lastbilen för att ge den ett namn. Om du vill att dina robotar ska använda en speciell anteckningsbok måste du lägga en kopia av den anteckningsboken i lastbilen. För att ge ett namn till en applet som skapats av en bild (med robotar på baksidan) lägger du bara ett textkort på bildens baksida.

En applet kan be användaren att lägga in värden. Anta att du översätter något till Java med ett fågelbo i en låda (och fågeln som tillhör det boet inte blir översatt till Java) och du har satt en etikett under facket i lådan. När då denna Java applet behöver ett värde och hittar fågelboet kommer den att be användaren om ett värde genom att skriva etiketten. På samma sätt är det med en fågel som är i ett fack med etikett och som får något. Då skrivs etiketten med en beskrivning av den sak fågeln fått. Tanken är att fåglar och bon används för meddelanden, i bägge riktningarna, mellan användaren och den Java applet som är aktuell.

Denna möjlighet kräver en webbläsare som klarar Java, t ex Microsoft Internet Explorer 4.0 (som finns på ToonTalk CD-ROM eller kan laddas ner från www.microsoft.com/ie) och Netscape Navigator 4.0 (som kan laddas ner från www.netscape.com). Kompilatorn för Java till byte fungerar bara i Windows 95 eller NT. Det innebär att ToonTalk för Windows 3.1 kan skapa Java, men att du behöver kompilera den på något sätt. (Du kan använda en gratis kompilatortjänst på nätet.)

Microsoft Internet Explorer (version 4.0 SP1) (som ingår i den första utgåvan av Windows 98) ger ibland en "Invalid Page Fault" när man försöker köra en Java applet som skapats av ToonTalk. Ibland hänger sig webbläsaren eller ger besked om ett Java error. Du kan markera "Internet Options" under "View" i menyraden, klicka på Advanced fliken och rulla dig ner till to "Java JIT compiler enabled". Var noga med att krysset tas bort. Detta bör lösa problemet men gör samtidigt alla Java applets långsammare. Denna bugg i Internet Explorer har rapporterats till Microsoft. Rapportera gärna alla problem med JIT frånslagen.
