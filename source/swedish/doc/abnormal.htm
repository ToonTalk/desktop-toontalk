Om ToonTalk plötsligt kraschar

Om ToonTalk plötsligt kraschar finns det flera tänkbara orsaker. Börja med att kolla om du kan göra samma sak igen för att få det att krascha.

  • ToonTalk har inte installerats på rätt sätt. Detta är inte särskilt troligt om ToonTalk fungerat bra tidigare. Att installera om går naturligtvis bra.
  • ToonTalk har för lite minne eller hårddiskutrymme. Kontrollera att din maskin har tillräckligt med minne och hårddiskutrymme.
  • Du har kanske flera versioner av ToonTalk. Klicka på "Set Options" när du startat ToonTalk och om du får frågan vilken version du vill köra kan du prova att byta version för att se om det hjälper.
  • ToonTalk kan köras antingen i fullskärmsläge eller i ett fönster. Pröva om problemet består i båda lägena. Klicka på "Set Options" när du startat ToonTalk för att ställa in detta.
  • 32 bitars versionen av ToonTalk använder sig av Microsofts DirectX speltillägg för Windows. Om du kör 32 bitars versionen måste du försäkra dig om att ha den senaste versionen av DirectX och dess drivrutiner. Mera information kan du få här www.microsoft.com/directx.
  • 16 bitars versionen kan få problem med en del äldre ljudkort. Prova med att stänga av ljudeffekter i "Set Options" när du startat ToonTalk.
  • På vissa datorer och ToonTalk-inställningar kan skärmen bli svart, börja "snöa" eller flimra. För att lösa det problemet kan du via Kontrollpanelen ändra Windows att använda 256 färger.

Kom ihåg att ToonTalk visar ett felmeddelande nästa gång det körs, om datorn stängdes av medan ToonTalk var igång. Du bör alltid avsluta ToonTalk genom att trycka på "Pause".

Rapportera gärna om problem som inte upphör genom att kontakta support@toontalk.com.
