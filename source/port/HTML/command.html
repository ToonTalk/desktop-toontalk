Opções de linha de comando do Toon Talk

Quando você inicia o Toon Talk, ele pergunta o seu nome. Você pode, em vez disso, fornecer seu nome na linha de comando. Você também pode clicar em "Set Options" (Definir opções) e alterar muitas das opções listadas abaixo muito facilmente. Quase todas as opções descritas abaixo podem ser controladas por itens da agenda de opções incluída na página 10 de sua agenda principal do Toon Talk.

Apenas para usuários avançados:

De forma alternativa você pode personalizar o Toon Talk através da edição do item de programa do Windows (Windows 3.1) ou do atalho (Windows 95) chamado "Toon Talk" (Alt-Enter quando o ícone é selecionado). Após a parte que vem no StartTT.exe, você pode acrescentar qualquer das seguintes opções separadas por espaços. Note que se o atalho termina com ", então coloque essas opções depois das aspas. Esses comandos irão cancelar quaisquer das opções selecionadas através do clique em "Set Options".

-n<name> (nome)
O nome pode ter qualquer tamanho, mas não deve conter espaços ou caracteres especiais. As primeiras oito letras do nome serão utilizadas como o nome do arquivo para o trabalho salvo. O nome "default" (padrão) é "No Name"( Sem Nome).Se você acrescentar isso, não será solicitado a inserir um nome de usuário.
-h (cabeça)
A cabeça pode ser 0 (careca), 1 (cabelo de menina) ou 2 (chapéu). O "default"é 2.
-s (som)
O som pode ser 0, para desligar o áudio. Se o sistema trava intermitentemente, isso pode se dever às más interações entre o driver de som e o Toon Talk. Veja se o 0 corrige o problema, e por favor informe-nos.
-f (tela cheia)
A tela cheia pode ser 0 (tela pequena 320x240), 1 (tela cheia com os gráficos do Windows fechados) ou 2 (640x480, Windows normal). A opção 1 tende a parecer um pouco mais bonita, e em algumas máquinas mais rápida, mas tem o problema de travar freqüentemente quando encontra um "bug". Algumas vezes, quando isso ocorre, a máquina precisa ser reiniciada. A opção 1 também funcionará a despeito da resolução da tela ou do número de cores. Também sob a opção 1 os comandos do Windows não funcionam. A opção 1 não funciona com alguns sistemas Windows NT.
-o
Tudo que é feito é gravado no arquivo. Pode ser recuperado por - i. É um grande meio de localizar um "bug". A extensão .DMO é acrescentada.
- i <input log file name>
Refaz todas as ações gravadas quando a opção -o foi utilizada. -I se comporta da mesma forma, mas reduzirá para a velocidade de gravação original se estiver rodando em uma máquina mais rápida.
-c (tamanho da cidade)
O tamanho "default" da cidade é de 10x10 blocos (cada bloco comporta 4 casas). O tamanho máximo da cidade é 255x255. Se negativo, iniciará a cidade com casas aleatórias. Seu computador precisa de muita memória e espaço livre em disco para rodar o Toon Talk com grandes cidades ativas.
-b <0,1,2 ou 3>
Descreve o número de botões do mouse normalmente disponíveis. O "default"é 1. Vide "botões do mouse" para detalhes. mouse buttons for details.
-r (percentagem de velocidade de leitura)
A velocidade de leitura controla quanto tempo o texto permanece em um balão de diálogo. Cem por cento é o "default". Números maiores fazem o texto permanecer por menos tempo.
-l (tamanho da letra nos balões de diálogo)
O "default"é 100 por cento.
-font_variable (nome da fonte variável)
Extensão variável da fonte utilizada nos balões de diálogo do Márcio e em outros lugares. Se seu nome tiver um espaço, como em "New Modern", digite como new_modern. Arial é o "default".
-font_fixed (nome da fonte fixa)
Extensão fixa da fonte utilizada em grupos de texto e números e em outros lugares.Courier New é o "default".
-m (memória cache).
O "default" é 100 (o qual utiliza 25% da memória livre). Números maiores deverão acelerar o sistema, mas deixarão menos memória para outras funções.
-u (perfil do usuário mantido)
O 1 significa manter o registro em *.usr. no qual as operações foram executadas e personalizar conforme a ajuda. A opção 0 o desativa. O "default" é 1.
-a (nível de conselho)
A escala varia de 0 a 6. Se estiver em 0, então Márcio dirá tudo o que é possível. Se estiver em 6, então ele não dirá nada. O "default" é 2.
-t (usuário temporário)
O 1 significa não ler ou gerar arquivos do usuário para a agenda. O "default" é 0.
-q (velocidade do Toon Talk)
Cem é o "default". Números maiores aceleram todas as animações.
--y< minimum frame duration in milliseconds> (duração mínima de quadros por milissegundos)
0 "default é 0.Cinqüenta, por exemplo, resultará em uma taxa máxima de 20 quadros por segundo, mesmo em um computador muito rápido.
-k
Se -k for 1 então os arquivos do Márcio e.log contêm mais informação acerca de quaisquer problemas encontrados. O "default"é 0.
-bird (veja os pássaros retornando de fora)
Se for 0, então os pássaros não despenderão tempo para retornar depois de entregar algo em um ninho quando o programador está observando de fora. O "default" é 1.
-j (rodando versão japonesa do Windows 3.1)
O 1 significa não exibir a tela de título. Parece funcionar para uma incompatibilidade entre o Win 3.1J e o WinG, ambos da Microsoft. Usualmente, a primeira vez que o texto é exibido, estará em branco.
-puzzle (desafio)
Inicia o Toon Talk rodando o desafio contido no nome do arquivo. O nome do arquivo também pode ser um número no qual o quebra-cabeça do sistema nomeado p.pzl é utilizado. Se você olhar no .log no subdiretório "Users" (Usuários), verá quais quebra-cabeças fez mais recentemente.
-next_puzzle 1 (próximo desafio)
Inicia o Toon Talk no próximo desafio neste nome de usuário. Ele começará com o primeiro desafio, se nenhum desafio foi resolvido por este usuário.
-autdemo <0,1 ou 2>
0 "default" é é 0 e desliga o modo auto demo. O 1 significa que, quando ocioso por 30 segundos, o sistema exibe repetidamente o "tour" introdutório. Se alguém começar a "play game" (jogar) ou "free play" (jogo livre), então ele esperará 120 segundos. Isto é controlado pela linha em seu toontalk.ini no diretório do Windows que lê o "AutoDemoMaxIdle=120". Se for -autdemo 2, a única diferença é que ele passará por todos os demos, um a um, quando ocioso.
-subtitles (velocidade das legendas)
Se houver a exibição de exemplos e legendas, então os exibirá a uma velocidade de leitura cujo "default" é 100. Se a velocidade é 0, então nenhum a legenda é exibida. A língua "default" das legendas pode ser alterada pelo "SubtitlesSuffix=ust" na seção "Executables" do ToonTalk.ini, onde UST é o sufixo de legendas para inglês americano, SET para sueco, e assim por diante.
-speak <0,1>
O 1 ativa a função texto-narração se estiver rodando na versão de 32 bits. Zero a desativa. O 1 é o "default" na versão de 32 bits e 0 na de 16 bits.
-balloon <0,1>
O 1 ativa os balões de palavras para o Márcio. O "default" é 1.
-tts_uses_ds <0,1>
O 1 significa que o programa de texto-narração utiliza DirectSound. Ele habilita simultaneamente a narração e os efeitos sonoros. O "default" começando com a versão 1.04 do ToonTalk é 1. (Antes dessa versão o Microsoft Speech API 3.0 era utilizado, e apresentou grandes atrasos.) Isto é apenas para a versão de 32 bits.
--mouse_sensitivity <sensitivity>
Cem é o normal.Um número maior que 100 fará com que pequenos movimentos do mouse produzam grandes movimentos no Toon Talk. Um número menor que 100 tornará o mouse menos sensível.
