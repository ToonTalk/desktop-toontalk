O que fazer se Márcio não fala no Toon Talk

Se você instalou o Toon Talk a partir de um CD-Rom e estiver rodando Windows 95, Windows 98 ou Windows NT 4.0, então Márcio deve ser capaz de falar. Aqui há algumas das coisas que podem estar erradas:

  1. Sem placa de áudio. Não é de surpreender se Márcio não falar se o sistema operacional não pode encontrar a placa de áudio.
  2. Drivers de áudio inadequados. O sistema operacional precisa de programas chamados drivers de áudio para rodar a placa de áudio. Os seus podem estar faltando, ser a versão errada ou muito antiga. Contate o seu fornecedor de informática para obter os últimos drivers de áudio.
  3. Placa de áudio de 8-bits. Se a sua versão do Toon Talk é anterior a 1.04 e sua placa de áudio é muito antiga, então pode ser que ela não seja capaz de operar com arquivos de som de 16 bits. Este problema foi corrigido pelo grupo de discussão da Microsoft em SAPI 4.0. Clique aqui para visitar o Web site do grupo de discussão da Microsoft para obter mais informações ou os últimos arquivos de discussão. A instalação do Microsoft SAPI 4.0 (Speech API) resolve esse problema.
  4. Uma alternativa. Márcio fala utilizando a comunicação Microsoft Speech API com os componentes DirectX da Microsoft. Se você estiver rodando a versão do Toon Talk 1.04, ou posterior, você pode tentar desligá-lo acrescentando "-tts_uses_ds 0" à linha de comando. Isto significará que você não ouvirá nenhum efeito sonoro enquanto Márcio estiver falando. Se você estiver rodando uma versão antes da 1.04, então tente "-tts_uses_ds 1" para ligá-la.
  5. Clique aqui para relatar um problema.
