Personalizando o ToonTalk

Na página 10 de seu caderno de anotações principal você encontrará um caderno de anotações de opções para personalizar seu ToonTalk. Ele funciona de maneira muito semelhante aos controladores de imagem e os sensores. Todos esses controles alteram elementos para o usuário atual, e não para todos os usuários. Você pode fazer alterações que afetam todos os usuários clicando em "Opções de Grupo" depois de iniciar o ToonTalk. (Os usuários mais avançados também podem personalizar o ToonTalk adicionando opções de linha de comando e editando o arquivo de inicialização ToonTalk .)

Somente para usuários avançados:

Os controladores de personalização no caderno de anotações de
opções são:

  • Página 2. Controla a aparência. Você pode selecionar ou apontar esse controlador para o objeto e pressionar a barra de espaço para selecionar a próxima opção. Suas opções são chapéu, careca ou cabelos longos roxos.
  • Página 4. Controla se o som está ativado. Pressionando a barra de espaço alterna-se entre "sim" ou "não" e pressionando-a novamente retorna-se para  "sim".
  • Página 6. Controla o tamanho da cidade. Varia de  1 a 255. Se for cidade negativa, será cerca de metade das casas. Este recurso não terá efeito até que você reinicie o ToonTalk. Seu computador precisa de muita memória e uma grande quantidade de espaço livre em disco para executar o ToonTalk com grandes cidades ativas.
  • Página 8. Uso do botão do mouse. Normalmente todos os botões do mouse realizam a mesma ação, exceto quando você está pilotando um helicóptero. Pode-se definir várias ações para os botões direito e do meio. Veja botões do mouse para obter mais detalhes.
  • Página 10. Controle do tamanho da tela. O padrão é hardware com tamanho de tela cheia. Pode ser 1x (320×240), 2x (640×480) até 4x. Se ele for configurado para um tamanho igual ou maior que o da tela, então é acionada uma versão soft do modo de tela cheia. Isto normalmente será mais lento e apresentará animações menos suavemente, mas em algumas máquinas é mais adequado. Em sistemas Windows NT anteriores à versão 4.0, o hardware com tamanho de tela cheia não funciona.
  • Página 12. Velocidade do ToonTalk. O padrão é 100. Se for mais de 100, todas as animações executarão rapidamente. Se for menor que 100, executarão mais vagarosamente. Usado no exemplo de Fibonacci.
  • Página 14. Duração mínima do quadro. O padrão é 0. Números mais elevados tornam o sistema lento, mas são mais adequados para obter-se uma melhor taxa de quadros.
  • Página 16. Tamanho da letra do Márcio . O padrão é 100. Números maiores fazem aumentar o tamanho das letras nos balões de texto.
  • Página 18. Velocidade da fala do Márcio. O padrão é 100. Números mais altos significam que os balões de sua fala demorarão mais para aparecer.
  • Página 20. Nível de fala do Márcio. O padrão é 2. Varia de 0 a 6. Se for 0, então Márcio falará qualquer coisa possível. Se for 6, então ele não dirá nada a menos que você esteja segurando ou apontando para algo quando chamá-lo.
  • Página 22. Controla se Márcio fala emitindo som, exibe suas palavras em balões ou ambos. Disponível somente na versão de 32 bits. A versão de 16 bits utiliza apenas balões de texto. O padrão na versão de 32 bits é a utilização de ambos. Veja também opção de linha de comando.
  • Página 24. Especifica em qual porta serial um controlador de Hyperbot está conectado. Os controladores de Hyperbot controlam um conjunto de brinquedos LEGO, Technic e Capsula. Disponível apenas na versão de 16 bits. Envie uma mensagem eletrônica para obter mais informações. A versão de 32 bits trabalha com comunicação baseada na porta serial.
